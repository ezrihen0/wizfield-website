# WIZFIELD Marketing Visual Asset Package v1

## 1. Executive Summary

This document is the canonical Marketing Visual Asset Package for the WIZFIELD marketing website.

It defines the major visual assets needed for:

- Homepage
- `/platform`
- `/ai`
- `/growth-center`
- `/language-store`
- `/multi-business`
- `/pricing`
- `/features`
- `/contact`

This is not an image-generation output, a Figma file, production code, a React component plan, or a website implementation. It is the visual asset specification layer that lets a designer, Figma agent, image-generation workflow, or implementation agent create and place the right visuals without re-deciding what they should depict.

The package inherits the locked visual thesis:

```text
Bright Operational Clarity
+ Glass Intelligence Surfaces
+ Cyan-to-Violet Digital Energy
+ Precision Typography
+ Controlled Premium Spacing
= WIZFIELD
```

`WIZFIELD Visual Brand DNA — Source of Truth.pdf` is the canonical visual authority for WIZFIELD. This asset package inherits that Brand DNA through the approved Website Visual Design System and does not use a substitute Brand DNA source.

## 2. Visual Asset Production Strategy

### A. Asset Types

Product UI Mockup: credible WIZFIELD interface compositions that show dashboard, calls, jobs, estimates, invoices, Growth Center, Language Store, workspace switching, pricing, or contact surfaces.

Diagram / Workflow Illustration: structured flow visuals that explain operating journeys, publish jobs, reviewed English output, workspace switching, or plan structure.

Hero Composite: larger composed visuals used in hero sections, combining product shell, micro UI cards, brand motif, ambient glow, and page-specific proof.

Micro UI Vignette: small product-detail cards that can be reused in grids, proof strips, or page-specific sections.

Brand Motif: W-derived node graphs, glow fields, dividers, CTA background motifs, and page background accents.

Card Illustration: contained visuals used inside feature, trust, boundary, qualification, or route cards.

Page Hero Visual: page-specific hero asset that introduces the visual thesis of one page.

Pricing / Comparison Visual: plan cards, plan model diagrams, comparison structures, Pro badge treatment, FAQ surface system.

Contact / Trust Visual: contact intent panels, inquiry cards, form treatment, response expectation panels, and success state visuals.

### B. Production Philosophy

WIZFIELD should prioritize credible product visuals, structured diagrams, reusable brand motifs, and light compositing. The best visual assets should feel like a real operating system made clearer by premium visual design.

Asset production should prefer:

- Product truth over decorative drama.
- Clear labels over vague tech abstraction.
- Reusable shells over one-off art.
- Bright surfaces over dark cyberpunk.
- System diagrams over random gradients.
- Motion-ready structure over noisy animation concepts.

Avoid vague pretty-tech art that does not explain product meaning.

### C. Static vs Motion-Aware Assets

Static assets:

- Pricing cards.
- FAQ surfaces.
- Contact qualification cards.
- Docs preview.
- Most dense product UI snippets.

Animation-ready assets:

- Hero composite layers.
- Ambient hero glow.
- W intelligence motif.
- Flow rails and nodes.
- Workspace switcher transition states.
- Growth Center publish flow.
- Language Store source-to-reviewed-output flow.

Light parallax capable:

- Homepage hero composite.
- Internal page hero visuals.
- Product mockup frames with floating cards.

Flow-line motion ready:

- Platform system map.
- Growth Center flow.
- Language Store split flow.
- Operational journey rail.
- Multi-business switcher flow.

Hover/lift compatible:

- Feature cards.
- Trust cards.
- Differentiator cards.
- Pricing cards.
- Contact intent cards.

### D. Reuse vs Unique Assets

Reusable systems:

- Glass product frame system.
- Flow rail / node system.
- W intelligence motif.
- Ambient glow library.
- Pricing card shell.
- Trust/boundary card shell.
- Workspace switcher shell.
- Docs preview surface.

Page-specific assets:

- Homepage hero operating-system composite.
- AI hero intelligence surface.
- Growth Center hero dashboard.
- Language Store hero split view.
- Multi-Business hero workspace switcher.
- Pricing hero plan structure cue.
- Contact hero sales-fit panel.

Desktop and mobile variants are required for large hero composites, complex system maps, flow diagrams, pricing comparison, and contact form treatments. Smaller card visuals can usually be produced as responsive compositions inside one master system.

## 3. Asset Taxonomy and Naming System

### A. Naming Rules

- Use lowercase kebab-case for asset IDs.
- Prefix asset IDs with `wf-`.
- Include page or shared surface context after the prefix.
- Name assets by function, not poetic mood.
- Avoid temporary names such as `cool-ai-graphic`, `gradient-thing`, or `hero-art-v2`.

Good examples:

- `wf-hero-operating-system-composite`
- `wf-platform-system-map`
- `wf-ai-business-brain-card`
- `wf-growth-signal-to-publish-flow`
- `wf-language-reviewed-english-flow`
- `wf-multibusiness-workspace-switcher`
- `wf-pricing-plan-structure-cards`

Every major asset below uses this required field order:

4. Primary Page Placement
5. Secondary Reuse Placement, if any
